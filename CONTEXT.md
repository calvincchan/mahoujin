# Context — Mahoujin

A mobile-first web app that transforms hand-drawn magic-circle photos into summoned creatures.

## Glossary

### Mahoujin
The hand-drawn magic circle (magic seal) the player photographs. Input to the whole pipeline.

### Creature Attributes
The structured JSON the analysis stage produces from a Mahoujin photo: `archetype` (closed enum), `element` (closed enum), `creatureName`, `description` (~100 words), `stats` (hp/mp/atk/def), `rarity` (1–5), `confidence` (high/low). Defined by `CreatureAttributesSchema` (Zod). The contract passed from analysis to summoning. `category` and `commonInspirations` are **not** on this contract — they are derived from the Archetype Registry by `archetype` key.

### Archetype
The creature's body blueprint (e.g. `fox`, `serpent`, `turtle`). One of a fixed set (closed enum). The LLM picks one during analysis.
_Avoid_: blueprint, name (the enum key is the Archetype; the registry's display label and the creature's given name are separate things).

### Category
The broad design family an Archetype belongs to (`Mammalian`, `Insectoid`, `Avian`, `Reptilian`, `Inorganic`, `Mythological`). A static property of each Archetype, **derived** from the Archetype Registry — never authored by the LLM.

### Creature Name
The individual creature's given name (e.g. "Frostcoil"), generated by the analysis LLM. Distinct from the Archetype key and the registry's display label.
_Avoid_: name (ambiguous — qualify as Creature Name).

### Element
The creature's elemental type — a closed enum of 19 Pokémon-style types (Normal, Fire, Water, … Stellar). Drives colour palette in image gen. Picked by the analysis LLM; an off-enum value fails validation and routes to the Mysterious fallback.
_Avoid_: type.

### Description
A short (~100 word) creative visual description of *this* creature's distinctive features, elemental variant, and personality, authored by the analysis LLM. Fed directly to the image-generation prompt as the primary visual anchor. Does **not** carry named-Pokémon references — those are owned by the Archetype Registry's `commonInspirations`, sampled into the image prompt.
_Avoid_: Trait (the field was renamed; "trait" in older PRD text meant a personality word, now folded into the Description prose).

### Archetype Registry
`src/lib/analyzer/archetypes.ts` — a curated table mapping each Archetype to its display label, Category, and a pool of `commonInspirations` (8–12 named Pokémon used as design nudges). At summon time, 1–2 inspirations are randomly **sampled** from the pool and injected into the image prompt, so each summon gets a coherent-but-varied silhouette nudge without the LLM ever naming a Pokémon. Evolutionary pattern is **not** tracked.

### Summoning
The stage that turns Creature Attributes into a rendered creature sprite via the image-generation prompt.

### Saved Creature
A kept creature persisted to the Crystal Shelf (IndexedDB). Extends Creature Attributes with `id`, `imageUrl`, `capturedAt`, a denormalized `category` (copied at save time so the shelf needn't re-derive), and `name` — the player's **final** name. `name` defaults to the LLM's `creatureName` but is overwritten if the player renames on Keep; `creatureName` preserves the original suggestion.
